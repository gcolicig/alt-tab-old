# Contributing

This fork values small, readable changes.

Good first areas:

- Improve fork setup and build scripts.
- Keep update, feedback, and crash-reporting behavior optional.
- Fix compatibility issues in newer macOS releases.
- Improve documentation around permissions and local development.

Before opening a pull request:

- Build locally with `./build.sh`.
- Keep changes focused.
- Do not add required external services without documenting how to disable them.
- Do not commit credentials, certificates, provisioning profiles, or private keys.
